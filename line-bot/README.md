# めぐり自転車 LINEボット セットアップ手順

Google Apps Script（GAS）で動く無料の自動応答ボット。`code.gs` を使う。

## できること

- リッチメニュー「写真を送る」タップ → 撮影ガイド（7枚）＋ボタン（📷写真を選ぶ／📸カメラで撮る／⚡電動アシスト）を自動返信
- 「電動」と送信 → 電動アシスト用の追加ガイド（バッテリー・充電器・鍵）を返信
- 写真を受信 → お礼メッセージを自動返信（10分に1回まで）
- それ以外のメッセージには反応しない（手動チャット対応の邪魔をしない）
- 写真を送ってくれたユーザーを記録し、10日後（`REVIEW_REQUEST_DELAY_DAYS`で調整可）にGoogleロコミ依頼を自動プッシュ送信
- 会話中に「キャンセル」「見送り」等のキーワードが出たら、そのユーザーへのレビュー依頼を自動的に取り消し（不成立の低評価対策）

## 費用

0円。応答メッセージは月200通の無料枠にカウントされない。GASも無料。

## セットアップ手順

### 1. Messaging APIを有効化

1. [LINE Official Account Manager](https://manager.line.biz/) にログイン
2. 設定 → Messaging API → 「Messaging APIを利用する」
3. プロバイダー名は「めぐり自転車」などで新規作成

### 2. チャネルアクセストークンを発行

1. [LINE Developers](https://developers.line.biz/console/) にLINEアカウントでログイン
2. 作成されたチャネルを開く → 「Messaging API設定」タブ
3. 一番下の「チャネルアクセストークン（長期）」で「発行」を押してコピー

### 3. GASプロジェクトを作成

1. [script.google.com](https://script.google.com/) → 「新しいプロジェクト」
2. `code.gs` の中身を全部貼り付け
3. 1行目付近の `CHANNEL_ACCESS_TOKEN = 'ここに...'` にコピーしたトークンを貼る
4. 右上「デプロイ」→「新しいデプロイ」→ 種類「ウェブアプリ」
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**全員**
5. 「デプロイ」→ 表示された **ウェブアプリのURL** をコピー

### 4. WebhookをLINEに登録

1. LINE Developers → チャネル → Messaging API設定タブ
2. 「Webhook URL」にGASのURLを貼り付けて「更新」→「検証」
3. 「Webhookの利用」をONにする

### 5. 応答設定を調整（重要）

LINE Official Account Manager → 設定 → 応答設定：

- チャット：**ON**（手動対応を残す）
- Webhook：**ON**
- 応答メッセージ：**OFF**（ボットと二重返信になるため。キーワード応答を設定していた場合は削除）

### 6. 動作テスト

自分のスマホで友だち追加したアカウントに対して：

1. リッチメニュー「写真を送る」をタップ → ガイドとボタン3つが返ってくる
2. 「⚡電動アシストの方はこちら」ボタン → 電動用ガイドが返ってくる
3. 「📷 写真を選ぶ」ボタン → スマホの写真選択が開く
4. 写真を送る → お礼メッセージが返ってくる

## レビュー依頼の自動送信（追加機能）

写真を送ってくれたユーザーをスプレッドシートに記録し、指定日数後にGoogleロコミ依頼をLINEでプッシュ送信します。

### 有効化の手順（すでにボットを運用中の場合、これだけ追加で行えばOK）

1. GASエディタで最新の `code.gs` の中身に差し替えて保存
2. Googleビジネスプロフィールが承認されたら、口コミ投稿リンク（「クチコミを増やす」で発行される短縮URL）を `GOOGLE_REVIEW_URL` に貼り付け
3. GASエディタ上部の関数選択プルダウンで `installReviewTrigger` を選び、▶実行（初回のみ・1回だけでOK）
   - 実行時に権限確認が出たら許可する
   - これで「毎日10時にsendReviewRequestsを実行する」トリガーが登録されます
4. 「デプロイ」→「デプロイを管理」→ 鉛筆アイコン → 新バージョンでデプロイ（コード変更を反映）

### 動作イメージ

- 顧客が写真を送る → 裏でスプレッドシート（`めぐり自転車_レビュー依頼キュー`、自動作成）に記録
- 10日後の毎日10時のトリガー実行時に、経過日数を満たしたユーザーへ自動でロコミ依頼メッセージを送信
- 送信済みのユーザーには二重送信されません
- 会話中に「キャンセル」「やめ」「見送り」「他で決め」「結構です」「不成立」「なしで」のいずれかが含まれるメッセージが来ると、そのユーザーの依頼は自動的に取り消されます（スプレッドシート上で`skipped`になります）。オーナー側の操作は不要です
- 送信タイミング（10日）は実際の引取までの平均日数に合わせて`REVIEW_REQUEST_DELAY_DAYS`を調整してください

## 文言を変えたいとき

`code.gs` の `replyPhotoGuide` / `replyEbikeGuide` / `thankForPhoto` 内のテキストを編集 →
保存 → 「デプロイ」→「デプロイを管理」→ 鉛筆アイコン → バージョン「新バージョン」→「デプロイ」。
（新バージョンでデプロイし直さないと反映されないので注意）

## 注意

- チャネルアクセストークンは秘密情報。GAS内にのみ貼り、Gitにはコミットしない
- カメラ／カメラロールのボタンはスマホ版LINEのみ表示（PC版では出ない）
